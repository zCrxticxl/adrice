# Security policy

## Reporting a vulnerability

Please do not create a public issue for a security vulnerability. Use the maintainer contact route listed in the repository profile, with affected desktop environment, script version, safe reproduction steps, and impact.

We aim to acknowledge reports within 7 days. Remove credentials, private URLs, and personal file paths from reports.

## Safety boundary

Imports, downloads, and theme changes must remain explicit, reversible, and constrained to documented targets. Any new network or shell-execution path requires validation and a safe failure mode.
